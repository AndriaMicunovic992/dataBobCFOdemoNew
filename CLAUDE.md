# DataBobIQ — Agent Context

A financial data analysis platform powered by AI. Users upload ERP/accounting
exports (xlsx/csv), an AI schema agent classifies columns, and a chat agent
helps build what-if scenarios with natural language.

## Tech Stack

| Layer    | Technology                                                    |
|----------|---------------------------------------------------------------|
| Backend  | Python 3.11+, FastAPI, SQLAlchemy 2 (async), Polars, Alembic |
| Frontend | React 18, Vite 5, React Query v5, Recharts, Lodash           |
| Database | PostgreSQL (asyncpg for app, psycopg2 for DDL/bulk)           |
| AI       | Anthropic Claude API (Sonnet for chat, Haiku for schema)      |
| Deploy   | Railway (Docker), local via docker-compose                    |

## Project Structure

```
backend/
  app/
    main.py              # FastAPI app, lifespan, CORS, static mount
    config.py            # Pydantic Settings (env vars)
    database.py          # Async + sync engines, Base, get_db dependency
    api/routes.py        # ALL endpoints (~2800 lines, single file)
    models/metadata.py   # SQLAlchemy ORM models (Model, Dataset, Scenario, etc.)
    schemas/api.py       # Pydantic request/response schemas
    services/
      chat.py            # Claude chat agent (tool-use, SSE streaming)
      schema_agent.py    # Claude schema detection agent (one-shot)
      ai_context.py      # XML context builder for AI agents
      scenario.py        # Polars-based scenario computation engine
      storage.py         # Dynamic table DDL + bulk data loading (sync engine)
      parser.py          # File parsing + type inference (Polars)
      transform.py       # Declarative data transformations
      calendar_svc.py    # Calendar dimension auto-generation
      query.py           # (stub) Dynamic query service
  alembic/versions/      # 0000–0010 migrations
frontend/
  src/
    App.jsx              # Entire UI (~4300 lines, single-file monolith)
    api.js               # API client (fetch wrapper + all endpoints)
    main.jsx             # React entry point
```

## Dev Commands

```bash
make install    # pip install -e . + npm install
make dev        # docker-compose postgres + alembic migrate + uvicorn --reload
make frontend   # vite dev server on :5173
make migrate    # alembic upgrade head
make reset-db   # alembic downgrade base + upgrade head
make clean      # stop docker + clear uploads
```

## Environment Variables

Set in `backend/.env` (copy from `.env.example`):

| Variable                 | Purpose                                              |
|--------------------------|------------------------------------------------------|
| `DATABASE_URL`           | PostgreSQL connection (auto-derived for async/sync)   |
| `ANTHROPIC_API_KEY`      | Fallback key if role-specific keys are absent         |
| `ANTHROPIC_API_KEY_CHAT` | Chat agent (scenario creation, data Q&A)              |
| `ANTHROPIC_API_KEY_AGENT`| Schema detection agent (column classification)        |
| `UPLOAD_DIR`             | File upload path (default: `./uploads`)               |
| `CORS_ORIGINS`           | Comma-separated allowed origins                       |

## Critical Rules

1. **Polars, not pandas** — all data processing uses Polars. Never introduce pandas.
2. **Sync engine for DDL** — `storage.py` uses `sync_engine` for table creation and bulk loads. App queries use `async_engine`.
3. **Dynamic tables use `ds_` prefix** — e.g., `ds_a1b2c3d4`. Generated by `storage.generate_table_name()`.
4. **Single-file frontend** — `App.jsx` is ~4300 lines. All components are inline functions. Styles use `S` and `C` objects (no CSS files).
5. **German + English** — data often has German column names and values. AI prompts handle both languages.
6. **Two separate AI agents** — schema agent (one-shot, Haiku) and chat agent (multi-turn tool-use, Sonnet). They use separate API keys.
7. **Route order matters** — in `routes.py`, literal paths (e.g., `/datasets/baseline`) must be registered before parameterized paths (`/datasets/{dataset_id}`).
8. **Token budget** — AI context builder targets <4000 tokens. Don't bloat system prompts.
